═══════════════════════════════════════════════════════════════════════════
                        KEY DESIGN DECISIONS
═══════════════════════════════════════════════════════════════════════════

1. Model Choice: all-mpnet-base-v2
   ✓ 768-dim embeddings
   ✓ State-of-art for semantic similarity
   ✗ NOT bert-base-nli-mean-tokens (deprecated)

2. Input Format: "[GENRES] x, y [DESC] text..."
   ✓ Combines genre signal + semantic signal
   ✓ Single embedding captures both
   ✗ Alternative: separate encoders (more complex)

3. L2 Normalization
   ✓ Makes cosine similarity = dot product
   ✓ Compatible with FAISS IndexFlatIP
   ✓ Math: cos(a,b) = a·b when ||a||=||b||=1

4. Genre Boost: 0.2 (20% of score)
   ✓ Balances semantic similarity + categorical signal
   ✓ Tunable in Sprint 3
   ✓ Formula: final_score = similarity + 0.2 * genre_overlap

5. Candidate Retrieval: 50 → re-rank to 10
   ✓ Gives genre filter room to promote relevant items
   ✓ Trade-off: more candidates = slower but better coverage

═══════════════════════════════════════════════════════════════════════════
                        KNOWN LIMITATIONS
═══════════════════════════════════════════════════════════════════════════

⚠️  Description Length Asymmetry
   • Books: ~523 chars avg, Movies: ~287 chars avg
   • Impact: Books get more semantic signal
   • Fix Sprint 3: Description normalization or separate encoders

⚠️  No Temporal Signal
   • No release date, popularity, recency
   • Impact: May recommend old/obscure movies
   • Fix Sprint 3: Add temporal features

⚠️  Cold Start Problem
   • New books require on-the-fly encoding
   • Impact: Slower for new items
   • Fix Sprint 3: Batch pre-compute

⚠️  Genre Mapping Quality
   • Depends on Sprint 1 Claude API mapping
   • Impact: Genre filter may be noisy
   • Fix Sprint 3: Manual review or learn end-to-end

⚠️  No Personalization
   • Pure content-based, no user history
   • Impact: Same book = same recommendations
   • Fix Sprint 4: Collaborative filtering
